# WOOHWA 상담실 예약 시스템 요구사항 정의서 (SRS v1.1)

**프로젝트명**: WOOHWA 상담실 예약 시스템
**작성자**: 희여 코치
**작성일**: 2026-04-17
**버전**: 1.2 (최종 결정사항 반영)
**브랜드**: WOOHWA — Transformational Moments

---

## 1. 프로젝트 개요

### 1.1 목적
명지전문대학교 연구동 2층 상담실 1·2·3의 이용 시간을 WOOHWA 코치들이 웹으로 간편하게 예약·확인할 수 있는 시스템을 구축한다.

### 1.2 배경
- 기존에는 상담실 예약이 수기/메신저로 관리되어 중복·현황 파악 어려움 발생
- 50~60대 코치 포함, 모바일에서 쉽게 쓸 수 있는 UI 필요
- 코치 이름·연락처·일정은 **대외비**이므로 외부 유출 방지 필요

### 1.3 주요 사용자
| 구분 | 설명 | 주요 기기 |
|------|------|-----------|
| 일반 사용자 (코치) | 50~60대 포함, 상담실 예약 주체 | 모바일 우선, PC 병행 |
| 관리자 | 희여 코치(본인) + 소수 지정 Gmail | PC 우선 |

### 1.4 브랜드 가이드
- **브랜드명**: WOOHWA (우화)
- **의미**: 번데기에서 나비로 변태(變態)하는 순간 → "Transformational Moments"
- **주 컬러**: WOOHWA Green (#9BC53D 계열, 정확한 hex는 디자인 단계에서 배너 이미지로부터 추출)
- **보조 컬러**: 연한 크림/베이지 (배경), 화이트
- **로고**: 번데기 심볼 + "WOOHWA" 워드마크

## 2. 시스템 구성

### 2.1 기술 스택
- **프론트엔드**: React + Vite (반응형, 모바일 우선)
- **백엔드/DB**: Google Apps Script + Google Spreadsheet
- **배포**: GitHub Pages 또는 Vercel (HTTPS 필수)
- **관리자 인증**: Google OAuth 2.0 (Gmail 화이트리스트)
- **저장소**: 희여 코치 **개인 Google 계정** (향후 운영 계정 이관 가능성 고려)

### 2.2 아키텍처 개요
```
[사용자 브라우저]
      ↓ HTTPS
[React 웹앱 (정적)]
      ↓ fetch (JSON)
[Google Apps Script (doPost/doGet)]
      ↓
[Google Spreadsheet (DB)]
```

### 2.3 Google Spreadsheet 시트 구성
| 시트명 | 용도 | 주요 컬럼 |
|--------|------|-----------|
| `Reservations` | 예약 데이터 | id, date, time, room, coachId, name, phone, carNumber, createdAt |
| `BlockedSlots` | 학교 지정 불가 시간 | id, type(공통/주차별), weekStartDate, dayOfWeek, time, room, reason |
| `Coaches` | 코치 명단 | id, name, phone, lastCarNumber, active |
| `AdminList` | 관리자 Gmail 화이트리스트 | email, role |
| `Settings` | 시스템 설정 | key, value (공통 비밀번호 해시 등) |
| `AccessLog` | 접근·예약 이력 (감사용) | timestamp, action, coachId, detail |

## 3. 기능 요구사항

### 3.1 사용자 진입 (인증)
- **FR-01** 웹 접속 시 **공통 비밀번호** 입력 화면 표시
  - **입력값은 평문으로 표시** (마스킹 없음) — 50~60대 사용자의 입력 확인 편의 우선
  - 초기 비밀번호: `woohwa` (관리자 화면에서 변경 가능)
- **FR-02** 비밀번호 통과 후 **코치 이름 선택** (드롭다운, `Coaches` 시트에서 로드)
- **FR-03** 동명이인 존재 시, 이름 선택 후 **연락처 뒷자리 4자리**로 본인 확인
- **FR-04** 선택된 코치 정보는 세션에 저장 (예약 자동 채움용)
- **FR-05** 비밀번호는 Apps Script 측에서 해시 비교 (평문 저장·클라이언트 노출 금지)
- **FR-06** 비밀번호 5회 연속 실패 시 10분간 입력 차단 (브라우저 단위)

### 3.2 예약 화면 (메인)
- **FR-10** 월~금 × 9시~20시(1시간 단위, 11개 슬롯) × 상담실 1/2/3 표 형태로 표시
- **FR-11** 상단 **주차 선택기** (◀ 이번 주 ▶)
  - 과거 주차 조회 불가
  - 미래 4주까지 이동 가능
- **FR-12** 슬롯 상태 표시:
  - 🟢 빈 슬롯 → 클릭 가능
  - 🟡 본인 예약 → "내 예약" 표시, 클릭 시 취소 메뉴
  - 🔴 타 예약자 → 이름만 표시 (차량번호 비공개)
  - ⚫ 학교 지정 불가 → "이용 불가" 표시, 클릭 불가
- **FR-13** 빈 슬롯 클릭 시 예약 모달 표시:
  - 이름 (자동 입력, 수정 불가)
  - 차량번호 (선택, **마지막 사용 번호 자동 입력**, 수정 가능)
  - 확인 / 취소
- **FR-14** **연속 슬롯 다중 선택 가능** (예: 10시·11시 동시 예약 → 레코드 2건)
- **FR-15** 동시 예약 충돌 시 Apps Script `LockService`로 순차 처리, 늦은 요청은 "이미 예약된 슬롯입니다" 안내
- **FR-16** 예약 완료 시 토스트 메시지 "예약이 완료되었습니다"
- **FR-17** 차량번호 등록 시 `Coaches` 시트의 `lastCarNumber` 필드 자동 업데이트

### 3.3 예약 취소
- **FR-20** 본인 예약 슬롯 클릭 시 "예약 취소" 버튼 표시
- **FR-21** 본인만 취소 가능 (세션 코치ID와 예약 coachId 일치 시)
- **FR-22** 취소 시 재확인 다이얼로그
- **FR-23** **예약 시작 시각 기준 1시간 전까지만** 취소 가능, 이후는 "취소 시한이 지났습니다" 안내
- **FR-24** 관리자는 시한 제약 없이 취소 가능

### 3.4 길안내 탭
- **FR-30** v1.0에서는 **플레이스홀더 페이지**로 구성 ("준비 중" 안내)
- **FR-31** 희여 코치가 추후 콘텐츠(텍스트/이미지/지도 링크) 직접 추가할 수 있도록 관리자 화면에 단순 편집 UI 제공
- **FR-32** 관리자는 마크다운 또는 리치텍스트로 길안내 내용 입력, 저장 시 즉시 반영

### 3.5 관리자 화면
- **FR-40** 관리자 진입: `/admin` URL → **Google OAuth 로그인** → `AdminList` Gmail 대조
- **FR-41** 관리자 대시보드
  - 오늘/이번주 예약 현황 카드
  - 예약 리스트 (날짜, 시간, 상담실, 이름, 연락처, 차량번호 전체 열람)
  - CSV 내보내기
  - 기간 필터 (이번 주 / 이번 달 / 사용자 지정)
  - 코치별 필터
- **FR-42** 학교 지정 불가 시간 관리
  - **공통 상시** (예: 매주 수요일 오후)
  - **특정 주차 한정** (예: 2026-04-20 주 월요일 9~10시)
  - 불가 사유 입력 (선택)
  - 일괄 등록 UI (여러 슬롯 체크 후 저장)
- **FR-43** 코치 명단 관리 (추가/수정/비활성화)
  - 필드: 이름, 연락처, 차량번호(선택)
- **FR-44** 공통 비밀번호 변경
- **FR-45** 관리자 Gmail 화이트리스트 관리
- **FR-46** 길안내 탭 콘텐츠 편집

### 3.6 데이터 관리
- **FR-50** 모든 예약·변경·취소 이력은 `AccessLog`에 기록
- **FR-51** 구글 스프레드시트는 희여 코치 개인 Google Drive 소유
- **FR-52** 관리자 외 스프레드시트 직접 공유 없음 (API 경유만)
- **FR-53** 알림 발송 기능은 v1.0에 포함하지 않음 (웹 내 확인만)

## 4. 비기능 요구사항

### 4.1 사용성 (Usability) — 최우선
- **NFR-01** 50~60대도 쉽게 쓸 수 있도록:
  - 버튼 최소 44×44px (터치 타겟)
  - 본문 글자 16px 이상
  - 색 대비 WCAG AA 이상
  - 한눈에 이해되는 아이콘 + 한글 텍스트 병기
- **NFR-02** 3탭 이내 예약 완료 (비번 → 이름/연락처 → 슬롯 클릭 → 확인)
- **NFR-03** 에러 메시지는 원인 + 다음 행동을 한 문장으로 안내

### 4.2 반응형
- **NFR-10** 모바일(375px~) / 태블릿 / PC(1920px) 대응
- **NFR-11** 모바일: 슬롯 표 세로 스크롤 / PC: 주 전체 한 화면

### 4.3 디자인 (WOOHWA 브랜드)
- **NFR-15** 메인 컬러: WOOHWA Green 계열 (배너 이미지 기준 #9BC53D 근사)
- **NFR-16** 상단 헤더에 WOOHWA 로고 배치
- **NFR-17** 타이포그래피: Noto Sans KR (가독성 우선)
- **NFR-18** 번데기/나비 모티프를 빈 상태(empty state), 로딩 등에 절제 있게 활용

### 4.4 보안
- **NFR-20** 모든 통신 HTTPS
- **NFR-21** 비밀번호는 Apps Script 측 해시 비교
- **NFR-22** 스프레드시트 직접 공유 금지 (API 경유만)
- **NFR-23** 관리자 인증은 Google OAuth + 화이트리스트 2중
- **NFR-24** 프론트엔드 번들에 비밀번호·스프레드시트 ID 포함 금지
- **NFR-25** Apps Script 배포: "본인으로 실행, 누구나 접근" + 서버 측 비번 검증
- **NFR-26** 코치 이름·연락처·차량번호는 검색엔진·캐시 노출 방지 (`noindex`, `no-store`)
- **NFR-27** 코치 연락처는 관리자만 열람 가능, 일반 사용자 화면에서는 본인 확인용 뒷자리 4자리만 사용

### 4.5 성능
- **NFR-30** 주간 달력 로딩 3초 이내
- **NFR-31** 예약 등록 2초 이내 완료

### 4.6 운영
- **NFR-40** Google Apps Script 무료 쿼터 내 운영
- **NFR-41** 주 1회 스프레드시트 자동 백업 (Apps Script 트리거로 복사본 생성)
- **NFR-42** 향후 운영 계정 이관 고려한 구조 (스프레드시트 ID·Script ID 환경변수화)

## 6. 화면 구성

### 6.1 사용자 플로우
```
[진입] 비밀번호 입력
   ↓
[이름 선택] 드롭다운 (동명이인이면 연락처 뒷자리 확인)
   ↓
[메인] 주간 예약 달력 (이번 주 기본)
   ├─ 탭: 예약하기 / 길안내
   ├─ 주차 선택기 (◀ 이번 주 ▶, +4주까지)
   └─ 슬롯 클릭 → 예약 모달 (차량번호 자동입력)
```

### 6.2 관리자 플로우
```
[진입] /admin
   ↓
[Google 로그인] OAuth → 화이트리스트 대조
   ↓
[대시보드]
   ├─ 예약 현황 · CSV 내보내기
   ├─ 불가 시간 관리 (공통/주차별)
   ├─ 코치 관리 (이름/연락처/차량번호)
   ├─ 길안내 콘텐츠 편집
   └─ 설정 (비밀번호·화이트리스트)
```

## 7. 확정된 결정사항 ✅

### 7.1 정책
- ✅ **사용자 인증**: 공통 비밀번호 + 이름 선택 (동명이인은 연락처 뒷자리 4자리)
- ✅ **초기 비밀번호**: `woohwa` (입력 시 평문 표시)
- ✅ **예약 정책**: 본인만 취소 가능, 시작 1시간 전까지
- ✅ **예약 단위**: 1시간, 여러 슬롯 동시 선택 가능
- ✅ **이동 범위**: 과거 조회 불가 / 미래 4주
- ✅ **일반 사용자 노출**: 이름만 (차량번호 비공개)
- ✅ **하루 최대 예약**: 제한 없음
- ✅ **차량번호**: 마지막 입력값 자동 기억, 예약 시 자동 채움

### 7.2 데이터/운영
- ✅ **저장소**: 희여 코치 개인 Google 계정 (추후 이관 가능성 고려)
- ✅ **코치 명단 컬럼**: 이름 / 연락처 / 차량번호(선택)
- ✅ **관리자 인증**: Google OAuth + Gmail 화이트리스트 (Gmail은 스프레드시트 배포 시 직접 입력)
- ✅ **알림**: v1.0은 웹에서만 확인 (이메일·카톡 알림 없음)
- ✅ **길안내 탭**: v1.0은 플레이스홀더, 관리자가 추후 콘텐츠 입력

### 7.3 디자인
- ✅ **브랜드**: WOOHWA (우화), 번데기→나비 모티프
- ✅ **메인 컬러**: WOOHWA Green (배너 이미지에서 직접 추출)
- ✅ **브랜드 가이드 문서 없음**: 배너 이미지 기반으로 디자인 시스템 구성
